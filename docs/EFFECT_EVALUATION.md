# 사업 효과 평가 구조

## 현재 웹 평가 모델

사업소관부서 지원도구는 지도에 배치한 사업을 계획안으로 관리하고,
`사업 적용·효과 계산` 버튼을 누른 시점의 수량과 효과를 대시보드에 반영합니다.

온도 저감 효과는 제공된 LH 계산 엔진의 개념을 웹 미리보기용으로 단순화하여 계산합니다.

1. 그림자 길이: `높이 / tan(태양고도)`
2. 단위 사업 유효 그늘: `(수관 또는 시설 면적 + 폭 × 그림자 길이) × (1 - 일사 투과율)`
3. 차단 에너지: 시군구별 09·12·15시 일사량과 유효 그늘 면적을 6시간 사다리꼴 적분
4. 기온 저감: `k_temp × (1 - albedo) × 일사량 × 평가면적 대비 유효 그늘 비율`

시군구 일사량은 `shared/data/climate/solar_admin_centroid_mean.csv`를 `SIG_CD`로 조회합니다.
태양고도는 `shared/data/climate/solar_altitude_by_sigungu.csv`, 평가면적은
`shared/data/administrative-regions/region_assessment_defaults.csv`에서 자동 조회합니다.
사용자는 두 값을 직접 입력하지 않습니다.

직접 태양고도 자료가 없는 상위 행정구역은 해당 지역명 아래의 하위 행정구역
태양고도를 평균하여 사용합니다. 예를 들어 `경기도 수원시(41110)`는 장안구,
권선구, 팔달구, 영통구의 값을 평균합니다.

## 적응 사업 디자인 흐름

1. 사용자가 사업 제목, 부문, 세부항목, 목표 물량과 점형·선형·면형을 정의합니다.
2. 디자인 완료된 사업은 왼쪽 사업 목록에 추가됩니다.
3. 사업을 선택하고 지도에서 점·선·면 데이터를 생성합니다.
4. `사업 적용·효과 계산`을 실행하면 현재 생성 물량이 사업이행평가에 반영됩니다.
5. 사업 부문에 연결된 효과 모듈이 적응효과평가에 결과를 표시합니다.

현재 건강 부문은 기온 저감 효과 모듈에 연결되어 있습니다.

### 가로수 기본 조건

상세 조건 입력 기능을 추가하기 전까지 가로수 사업에는 다음 기본값을 자동 적용합니다.

- 수관폭: `4m`
- 수목 높이: `6m`
- 일사 투과율: `15%`
- 수관 형태: 원형
- 지역 태양고도 자료가 없을 때: `45°`
- 평가면적 자료가 없을 때: `10,000㎡`

지도에 가로수 점을 배치한 경우 실제 배치 개수를 계산에 사용합니다. 아직 배치하지 않은
상태에서 효과 계산을 실행하면 사업 디자인의 목표 물량을 시범 적용량으로 사용합니다.

일사량은 제공 엔진의 `solar_admin_centroid_mean.csv`를 그대로 사용합니다. 선택 시군구의
중심점 주변 반경 `1,000m`에 포함되는 KMAP 셀을 평균한 09·12·15시 수평면 일사량입니다.
직접값이 없는 상위 행정구역은 하위 시군구 값을 평균하며, 일사량 자료가 전혀 없는
지역은 임의 기본값으로 계산하지 않습니다.

## 정밀 계산 API 연결 지점

현재 브라우저 모델은 사업 배치안 비교를 위한 빠른 미리보기입니다. 정밀 분석 시에는
제공 엔진의 다음 기능을 공통 Python API로 연결합니다.

- `canopy_geometry.py`: 수종·규격 기반 수관폭 보정
- `shadow.py`: 실제 수목 위치와 그림자 폴리곤 중첩 계산
- `shadow_frac.py`: 5m 등 표준격자별 그림자 비율 계산
- `temper_calc.py`: 포장 알베도, LAI, 지역 일사량을 포함한 기온 저감 계산

## 효과 모듈 확장

웹 계산 함수는 사업 목록을 배열로 입력받도록 구성되어 있어 가로수·그늘막 외 사업을
추가할 수 있습니다. 향후 동일한 대시보드에 탄소흡수, 미세먼지 저감, 유출저감 계산
모듈을 추가하고 사업별 적용 가능 효과를 등록하는 구조로 확장합니다.
