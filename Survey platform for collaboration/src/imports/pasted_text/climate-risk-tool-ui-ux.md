공무원 대상 기후변화 리스크 설문·평가 웹도구의 UI/UX 화면 초안을 설계해 주세요.

이 웹도구는 지자체 기후대응 주관부서가 지역 기후변화 리스크 목록을 작성하고, 관련 부서에 설문을 배정한 뒤, 각 부서 담당자들이 리스크별 질문에 응답하고, 주관부서가 응답 결과를 확인하여 최종 리스크 우선순위와 시급성을 정리하는 협업형 플랫폼입니다.

전체 서비스는 공공기관 업무 시스템처럼 신뢰감 있고 명확해야 하며, 과도하게 화려한 디자인보다는 행정업무에 적합한 차분한 디자인을 원합니다. 색상은 흰색 배경, 짙은 남색 또는 청록색 계열의 포인트 컬러, 연한 회색 구분선을 사용해 주세요. 사용자는 공무원이므로 정보 구조가 명확하고, 표, 상태 배지, 카드, 사이드패널, 지도 영역이 잘 구분되어야 합니다.

핵심 사용자 유형은 두 가지입니다.

1. 주관부서 관리자
- 지자체 기후대응 부서 직원
- 설문 프로젝트를 생성함
- 지역 리스크 목록을 작성함
- 리스크별 현황정보와 지도 레이어를 설정함
- 관련 부서에 리스크를 배정함
- 응답 현황을 확인함
- 최빈 응답값과 동률 여부를 확인함
- 최종 확정값과 주관부서 의견을 입력함
- 결과를 엑셀로 다운로드함

2. 응답부서 담당자
- 각 부서의 개인 참여자
- 자신이 속한 부서에 배정된 리스크 목록만 확인함
- 리스크를 클릭하면 지도, 현황정보, 설문 질문을 한 화면에서 확인함
- 질문 1~3번에 응답하고 임시저장 또는 제출함

이 서비스는 프로젝트 단위로 운영됩니다.
예: “수원시 2026 기후변화 리스크 설문 프로젝트”, “인천광역시 2026 기후변화 리스크 설문 프로젝트”

지역 리스크는 국가 리스크에서 자동 선택하는 것이 아니라, 주관부서가 직접 작성합니다. 다만 국가 리스크는 참고자료로 볼 수 있어야 합니다.

리스크 분류 구조는 다음과 같습니다.
- 부문
- 세부부문
- 세세부태그

부문 예시는 다음과 같습니다.
- 국토
- 연안
- 물관리
- 생태계/산림
- 건강
- 산업 및 에너지

세부부문 예시는 다음과 같습니다.
- 생태계/산림 > 산림
- 생태계/산림 > 생태계
- 물관리 > 치수
- 물관리 > 이수
- 건강 > 감염병
- 건강 > 건강질환

세세부태그는 선택값입니다.
예:
- 산림 > 병해충
- 산림 > 산림재해
- 산림 > 생산성
- 감염병 > 수인성 매개질환
- 감염병 > 곤충 및 설치류 매개질환

화면은 최소 6개를 설계해 주세요.

[화면 1] 주관부서 관리자 대시보드
- 상단에는 프로젝트명, 지자체명, 설문 기간, 전체 진행률을 표시합니다.
- 주요 카드 4개를 배치합니다.
  1) 등록된 지역 리스크 수
  2) 배정된 부서 수
  3) 제출 완료 응답 수
  4) 확정 완료 리스크 수
- 아래에는 부문별 진행현황 막대 또는 카드가 있어야 합니다.
- 우측 또는 하단에는 최근 제출된 응답, 수정요청, 확정대기 리스크 목록을 보여주세요.
- 주요 버튼: “지역 리스크 관리”, “부서 배정 관리”, “응답 결과 확인”, “엑셀 다운로드”

[화면 2] 지역 리스크 작성/관리 화면
- 좌측에는 부문 필터가 있습니다.
  예: 전체, 국토, 연안, 물관리, 생태계/산림, 건강, 산업 및 에너지
- 중앙에는 지역 리스크 목록 테이블이 있습니다.
- 테이블 컬럼:
  - 리스크명
  - 부문
  - 세부부문
  - 세세부태그
  - 연결된 현황정보 수
  - 연결된 지도 레이어 수
  - 배정 부서 수
  - 상태
  - 수정 버튼
- 우측에는 선택한 리스크의 상세 편집 패널이 열립니다.
- 상세 편집 패널에는 다음 입력 항목이 있습니다.
  - 지역 리스크명
  - 리스크 설명
  - 부문 선택
  - 세부부문 선택
  - 세세부태그 선택
  - 국가 리스크 참고자료 선택
  - 응답자에게 보여줄 안내문
  - 현황정보 페이지 구성 버튼
  - 지도 레이어 선택 버튼
  - 저장 버튼

[화면 3] 현황정보 페이지 구성 화면
- 이 화면은 주관부서가 응답자에게 보여줄 리스크 관련 현황정보를 구성하는 화면입니다.
- 좌측에는 콘텐츠 블록 추가 메뉴가 있습니다.
  - 제목
  - 본문
  - 표
  - 그래프
  - 지도 이미지
  - 주석
  - 출처
- 중앙에는 보고서형 미리보기 영역이 있습니다.
- 예시 콘텐츠는 “건강 부문 현황”으로 구성해 주세요.
- 예시 내용:
  - 제목: 건강 부문 현황
  - 설명: 인천광역시의 건강부문 관련 현황은 아래와 같습니다.
  - 소제목: 수인성 매개 질환에 대한 건강 취약성
  - 본문: 수인성 매개질환으로 인한 질병 중 일부 감염병은 최근 다시 증가하는 추세입니다.
  - 그래프 카드 2개
  - 지도 이미지 카드 2개
  - 그림 캡션
- 우측에는 선택한 블록의 속성 편집 패널이 있습니다.
  - 제목
  - 본문
  - 이미지 업로드
  - 자료 출처
  - 캡션
  - 순서 변경
- 상단에는 “미리보기”, “저장”, “응답화면에서 보기” 버튼이 있습니다.

[화면 4] 리스크-부서 배정 화면
- 좌측에는 지역 리스크 목록이 있습니다.
- 중앙에는 선택한 리스크의 기본정보 카드가 있습니다.
  - 리스크명
  - 부문/세부부문/세세부태그
  - 설명
- 우측에는 관련 부서 선택 영역이 있습니다.
- 부서 목록은 체크박스 형태로 표시합니다.
  예:
  - 기후대응과
  - 도시계획과
  - 하천관리과
  - 재난안전과
  - 건강증진과
  - 보건정책과
  - 산림녹지과
- 하나의 리스크에 여러 부서를 배정할 수 있어야 합니다.
- 하단에는 배정 현황 테이블이 있습니다.
  - 리스크명
  - 배정 부서
  - 응답자 수
  - 제출 수
  - 마감일
  - 상태
- 주요 버튼: “선택 부서 배정”, “마감일 설정”, “배정 저장”

[화면 5] 응답자 리스크 목록 화면
- 응답부서 담당자가 로그인했을 때 보이는 화면입니다.
- 상단에 사용자의 부서명과 프로젝트명이 표시됩니다.
  예: “건강증진과 / 인천광역시 2026 기후변화 리스크 설문”
- 본문에는 본인 부서에 배정된 리스크 목록만 카드 또는 테이블로 표시합니다.
- 각 리스크 카드에는 다음 정보가 있어야 합니다.
  - 리스크명
  - 부문/세부부문/세세부태그
  - 응답 상태
  - 제출 마감일
  - 현황정보 요약
  - “응답하기” 버튼
- 상태 배지는 다음을 사용합니다.
  - 미작성
  - 작성중
  - 제출완료
  - 수정요청
  - 확정

[화면 6] 지도 + 현황정보 + 설문 응답 화면
- 가장 중요한 화면입니다.
- 화면은 크게 지도 영역과 사이드패널로 구성합니다.
- 좌측 또는 중앙에는 지도 영역이 있습니다.
  - 해당 지자체의 지도 경계가 표시됩니다.
  - 선택된 부문 또는 리스크에 맞는 지도 레이어가 표시됩니다.
  - 지도 위에는 레이어 토글 패널이 있습니다.
  - 예: 폭염일수, 고령인구, 감염병 발생 현황, 침수흔적도, 산림병해충 발생지역
- 우측에는 사이드패널이 있습니다.
- 사이드패널은 탭 구조로 구성합니다.
  1) 리스크 개요
  2) 현황정보
  3) 설문응답
- 리스크 개요 탭에는 리스크명, 부문, 세부부문, 설명, 응답 안내문을 표시합니다.
- 현황정보 탭에는 주관부서가 구성한 보고서형 현황정보를 표시합니다.
  - 제목
  - 본문
  - 그래프 카드
  - 지도 이미지
  - 표
  - 출처
- 설문응답 탭에는 질문 1~3번이 구조화되어 표시됩니다.

질문 1:
“시급성을 고려하여 위의 리스크가 해당하는 리스크 유형을 한 개만 고르고, 해당 리스크의 시급성을 선택해 주십시오.”
- 표 형태로 구성
- 행: 리스크 유형
  1) 우선적 추가조치 필요한 리스크 항목
  2) 장기적 연구 및 모니터링 필요한 리스크 항목
  3) 잠재적 영향이 존재하는 리스크
- 열: 시급성
  1) 시급성 낮음
  2) 시급함
  3) 매우 시급함
- 응답자는 9개 칸 중 하나만 선택할 수 있습니다.

질문 2:
“리스크가 발생할 가능성이 큰 지역 공간에 대한 질문입니다. 위의 리스크가 발생 가능한 지역의 범위와 발생 우려가 큰 공간을 선택해 주십시오.”
- 표 형태로 구성
- 행: 토지이용 유형
  1) 시가화·건조지역
  2) 농업지역
  3) 산림, 초지 및 습지
  4) 나지
  5) 수역 및 인근지역
  6) 복합지역
- 열: 공간 범위
  1) 지자체 전역
  2) 특정 권역
  3) 특정 시군구 또는 읍면동
  4) 기타
- 복수 선택이 가능해야 합니다.

질문 3:
“위의 리스크가 향후 기후변화가 진행될 경우 각 시간범위별로 미치는 영향에 어느 정도 변화가 있을 것으로 예상되는지 표시해 주십시오.”
- 표 형태로 구성
- 행: 시간 범위
  1) 단기간, 5년 이내
  2) 장기간, 5년 이후
- 열: 영향 변화 예측
  1) 감소
  2) 현재와 비슷
  3) 증가
- 각 행에서 하나씩 선택할 수 있습니다.

설문응답 탭 하단에는 다음 버튼이 있습니다.
- 임시저장
- 제출하기
- 이전 리스크
- 다음 리스크

[화면 7] 주관부서 응답 결과 검토 화면
- 주관부서가 부서별·개인별 응답 결과를 확인하는 화면입니다.
- 상단에는 리스크명과 부문 정보가 표시됩니다.
- 좌측에는 질문별 최빈값 요약 카드가 있습니다.
  - 질문 1 대표 응답
  - 질문 2 대표 응답
  - 질문 3 단기 대표 응답
  - 질문 3 장기 대표 응답
- 동률이 발생한 경우 “동률 발생” 배지를 표시하고, 동률 응답값을 모두 보여주세요.
- 중앙에는 응답 분포 차트를 표시합니다.
  - 선택지별 응답자 수
  - 부서별 응답 분포
- 하단에는 개인 응답 목록 테이블을 표시합니다.
  - 응답자
  - 근무부서
  - 질문 1 응답
  - 질문 2 응답
  - 질문 3 응답
  - 제출일시
- 우측에는 주관부서 최종 확정 패널이 있습니다.
  - 시스템 추천 대표값
  - 최종 확정값 선택
  - 주관부서 의견 입력
  - 확정 버튼
  - 수정요청 버튼

[화면 8] 결과 요약 및 엑셀 다운로드 화면
- 전체 프로젝트 결과를 확인하는 화면입니다.
- 상단에는 전체 요약 카드가 있습니다.
  - 전체 리스크 수
  - 응답 완료율
  - 확정 완료율
  - 매우 시급 리스크 수
- 중앙에는 부문별 우선순위 표가 있습니다.
- 표 컬럼:
  - 순위
  - 리스크명
  - 부문
  - 세부부문
  - 세세부태그
  - 대표 시급성
  - 주요 공간범위
  - 단기 영향
  - 장기 영향
  - 확정 여부
- 좌측에는 부문 필터가 있습니다.
- 우측 상단에는 “전체 결과 엑셀 다운로드” 버튼이 있습니다.
- 엑셀 다운로드 설명 문구:
  “다운로드 파일에는 전체 우선순위, 부문별 응답현황, 리스크별 상세응답, 공간범위 선택 요약이 포함됩니다.”

디자인 요구사항:
- 공공기관 업무 시스템 느낌
- 과도한 애니메이션 없이 명확하고 안정적인 레이아웃
- 표와 카드 중심의 정보 구조
- 지도와 사이드패널이 잘 구분되는 화면
